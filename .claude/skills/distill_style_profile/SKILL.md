# distill_style_profile

公開済みレポート群から文体プロファイルを蒸留します。日々のレビューがコーパス全体を読む代わりに、このスキルが定期的に傾向を凝縮して `guidelines/style_profile.md` に反映します。

## 実施タイミングの目安

- 初回: `corpus/` に5本程度たまったとき
- 以降: 公開版が5〜10本増えるごと、または文体の変化を感じたときに任意で

## 手順

1. **対象の収集**: `corpus/*.md` と `reviews/*/final.md` を `published_at` 順に列挙する。10本を超える場合は、直近8本＋古い時期の代表2本程度に絞る（直近の文体を重視しつつ、一貫した癖を見分けるため）。
2. **蒸留**: [guidelines/style_profile.md](../../../guidelines/style_profile.md) のセクション構成（全体構成 / 語り口・文体 / 語彙・表現の癖 / 写真の使い方 / 数値・固有名詞の扱い）に沿って傾向を抽出する。
   - 書くのは**複数のレポートに共通して現れる癖**だけ。1本にしか現れない特徴は書かない。
   - 各項目には可能な範囲で実例（レポートからの短い引用）を1つ添える。
3. **更新案の提示**: style_profile.md の現状との差分を before/after で提示し、ユーザーの承認を求める。
4. **反映**: 承認された内容だけを Edit で反映する。冒頭の「状態: 未蒸留」ブロックは初回蒸留時に削除し、末尾の蒸留記録に「蒸留日（JST）・対象本数・対象期間」を追記する。承認されなければ変更しない。
