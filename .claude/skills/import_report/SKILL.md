# import_report

過去レポートを1件ずつ `corpus/` に取り込みます。本システム運用開始後の新しいレポートは `reviews/*/final.md` がコーパスを兼ねるため、このスキルの対象は運用開始前の過去レポートです。管理上の基準日は公開日ではなく**山行日**とします。

入力はユーザーの貼り付けテキストを前提とし、**山行日・タイトル・本文**の3項目を必須とします。いずれか1つでも不足している場合は処理を進めず、ユーザーに不足項目の提示を依頼します。

## 手順

1. **入力確認**: ユーザーから山行日・タイトル・本文を受け取る。いずれかが不足している場合は、不足項目を明示して追加入力を依頼する。補助情報として元URLを受け取ってもよいが、必須ではない。
2. **Markdown 変換**: [docs/text_to_markdown.md](../../../docs/text_to_markdown.md) に従い、**語句を変えずに**変換する。frontmatter は `title` / `published_at` / `created_at` を基本とし、元URLも渡されている場合のみ `source_url` を追加する。`published_at` には山行日を入れる。
3. **保存**: `corpus/YYYY-MM-DD_<slug>.md`（`YYYY-MM-DD` は山行日）として Write する。slug は山名等から候補を提案し、ユーザーに確認してから確定する。同名ファイルが既にある場合は上書きせず相談する。
4. **報告**: 保存したパスと、`corpus/` 内の登録済み件数を報告する。5本程度たまっていて文体プロファイルが未蒸留（または蒸留が古い）なら、`/distill_style_profile` の実施を案内する。
