# テキスト→Markdown 変換規則

`register_draft` / `register_gemini_review` / `register_final` / `import_report` が共通で従う変換規則です。

## 大原則

- **本文の語句を一切変えない**。誤字と思われる箇所もそのまま写す。この工程で行うのは構造の整形のみで、レビューや校正は行わない。
- 変換に迷う構造（見出しか強調か判別できない等）があれば、推測で確定せずユーザーに確認する。

## 変換規則

1. **見出し**: レポートのセクションタイトルにあたる行は `##`、その下位の小見出しは `###` にする。文書タイトルは frontmatter の `title` に入れ、本文には `#` を置かない。
2. **段落**: 空行区切りの段落構造を維持する。元テキストの単一改行が「段落内の改行」か「段落の切れ目」か判別しづらい場合は前後の文脈で判断し、それでも迷う場合は段落の切れ目として扱う。
3. **箇条書き**: 行頭の「・」「-」「*」などは `-` に統一する。
4. **写真**: 写真が入る位置が読み取れる場合は `[写真: 簡単な説明]` のプレースホルダ行として残す。公開URLから取り込む場合で画像URLが取れるときは `[写真: 説明](URL)` 形式でもよい（画像バイナリは取り込まない）。
5. **表・その他の構造**: Markdown で素直に表現できる場合のみ変換し、無理な場合はプレーンテキストのまま残す。

## frontmatter

各ファイルの冒頭に YAML frontmatter を付与する。必須項目はスキルごとに異なる（各 SKILL.md を参照）。共通の項目は以下。

```yaml
---
title: レポートのタイトル
created_at: 2026-07-08        # 登録日（JST）
published_at: 2026-07-05      # 公開日（公開版・corpus のみ）
source_url: https://...       # 公開URLから取得した場合のみ
---
```
